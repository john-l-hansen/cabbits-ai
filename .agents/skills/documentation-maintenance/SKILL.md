# Documentation Maintenance Skill

When implementation changes affect product behavior, update the relevant files in /docs and /specs.

Process:
1. Identify which behavior changed.
2. Update only the relevant documentation.
3. Keep docs concise and source-like.
4. Do not invent future features.
